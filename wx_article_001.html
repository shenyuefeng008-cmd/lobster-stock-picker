一、这个系统是怎么来的

三年前我开始做量化交易研究。不是那种「写几行Python调个接口就能赚钱」的天真想法——我知道那是忽悠人的。

我真正想做的是：把交易从「玄学」变成「工程」。

大多数人的交易是这样的：

  • 看到一个涨停 → 追进去 → 被套
  • 看到一个消息 → 冲进去 → 被套
  • 看到一个大V推荐 → 买进去 → 被套
  • 亏损了 → 加仓摊平 → 被套更深

有没有一种可能：把选股逻辑写进代码，让机器替你执行纪律？

人最大的问题是管不住自己。但代码不会。

代码不会因为今天赚了钱就变得贪婪，不会因为昨天亏了钱就变得恐惧。它会老老实实按照规则判断：这个条件成立吗？成立就买，不成立就不买。

这就是龙虾系统的出发点。

二、龙虾选股的三维框架

系统核心：点·线·面 三维度框架。

维度 名称 核心逻辑 适用情绪
1.0 点（打板） 一进二涨停接力 + 分歧低吸 <1500
2.0 线（板块轮动） 板块卡位 + 高低切换 2500-3500
3.0 面（容量趋势） 产业逻辑 + 趋势低吸 1500-2500

* 情绪依据：当天市场涨跌家数

核心思想很简单：不同市场环境，用不同的策略。

市场好的时候（涨的多），做追涨打板；市场差的时候（跌的多），做分歧低吸；市场不好不坏的时候，做趋势慢涨。

听起来很朴素对吧？但实际操作中，大多数人做反了：

  • 涨到高潮了，冲进去追板 —— 第二天高开低走吃面
  • 跌到冰点了，害怕割肉 —— 割在最低点

龙虾系统的核心能力就是：自动识别当天该用什么策略，并且强制自己执行。

代码不犹豫，代码不后悔。

三、风控：这个系统最贵的部分

你可能会觉得，一个选股系统最重要的是「选出涨停股」。

错了。

最重要的是「亏钱时及时止损」。

三年打磨，这个系统加了五层风控：

🛡️ 第一层：情绪过滤
涨跌家数低于1500 → 禁用追涨策略
涨跌家数高于3500 → 禁用大部分策略，强制休息
—— 市场不行的时候，少操作就是赚

🛡️ 第二层：仓位上限
冰点期（<1500）→ 最多5成仓
中性区（2000-2500）→ 最多9成仓
高潮期（>3500）→ 1-2成，辅助模式
—— 决不允许满仓赌

🛡️ 第三层：硬止损
单只股票亏损 ≤ -7% → 无条件卖出
—— 亏7%活的下来，亏30%活不下来

🛡️ 第四层：技术止损
跌破MA10 → 检查均线形态，确认破位则卖出
—— 不幻想"还能涨回来"

🛡️ 第五层：催化剂否决权
产业利好强度不足 → 即使技术面符合也放弃
—— 没有基本面支撑的涨停，是空中楼阁

这五层风控不是摆设，是写在代码里的——每天自动跑，自动检查，自动执行。

大多数散户亏钱，不是因为选股差，而是因为没有止损纪律。

这个号不吹"涨停率90%"，那都是骗人的。
