# L4 执行控制 — P0 最终否决权落地

## 时间
2026-05-24 23:25

## 目标
在 `simulated_trading.py` 中实现 L4 最终否决权（P0），阻止极端行情下的买入，并记录止损错误实现冷却机制。

## 实施内容

### 1. 新增四个函数（插入在 `calc_stock_factor()` 之前，原 L162 处）

| 函数 | 功能 |
|------|------|
| `_load_trade_errors()` | 加载 `trading/trade_errors.json` |
| `_save_trade_errors(data)` | 保存错误记录 |
| `record_trade_error(code, name, reason, loss_pct, sell_type)` | 止损/情绪卖出时记录错误，≥2次触发7天冷却 |
| `final_veto_check(code, name, price, dimension, up_count)` | L4最终否决权，返回 `(ok, reason)` |

### 2. `final_veto_check()` 五道否决条件

| # | 条件 | 动作 |
|---|------|------|
| 1 | 时间 14:55-15:00 | 否决（防尾盘偷袭） |
| 2 | `trade_errors.json` 中该股票在冷却期内 | 否决，提示冷却截止日期 |
| 3 | 上证指数跌幅 > 5% | 否决（市场异常） |
| 4 | 个股当日跌幅 > 9% | 否决（个股异常） |
| 5 | 个股成交额 < 5000万（字段57） | 否决（流动性不足） |

注：条件3/4/5 获取失败时静默放行（不因接口超时阻止正常交易）。

### 3. Hook 接入

- `buy()` L408：市价买入前调用 `final_veto_check()`，否决则直接 return 原因字符串
- `sell()` L~370：卖出成功后调用 `record_trade_error()`，止盈不记录
- `sell_partial()`：部分止盈，不记录错误（无需 hook）

### 4. 新建文件

`trading/trade_errors.json` — 初始结构：
```json
{
  "meta": {"description": "L4执行控制 — 交易错误记录与冷却", "version": "1.0"},
  "errors": [],
  "cooldown": {},
  "stats": {"total_errors": 0, "by_code": {}, "by_reason": {}}
}
```

## 已知限制

- `final_veto_check()` 的市值/成交额获取依赖 `qt.gtimg.cn` 接口，盘中超时会导致条件3/4/5静默放行
- 冷却期判断使用字符串比较（`YYYY-MM-DD`），跨时区需注意
- `record_trade_error()` 的冷却触发条件是「最近50条中止损≥2次」，非全局统计
