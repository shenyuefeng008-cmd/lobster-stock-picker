# 买点监控候选池读取bug修复

**时间**：2026-05-20 13:36
**问题**：买点监控报告"候选池为空"，但盘前选股引擎实际有输出

## 问题定位

候选池文件结构：
```json
{
  "date": "2026-05-20",
  "candidates": {
    "1.0一进二": [...],
    "1.0分歧低吸": [...],
    "2.0板块卡位": [...]
  }
}
```

买点监控脚本读取路径错误：
- 实际路径：`data['candidates']['1.0一进二']`
- 脚本读取：`data['1.0一进二']` → 读不到

## 修复内容

文件：`scripts/cron-tasks/CRON_BUYPOINT_TASK.md`

### 步骤2 移出检查代码（3处）

1. **候选收集**（line 127-135）
```python
# 修复前
for dim in ['1.0一进二', ...]:
    for item in data.get(dim, []):

# 修复后
candidates = data.get('candidates', data)  # 兼容两种格式
for dim in ['1.0一进二', ...]:
    for item in candidates.get(dim, []):
```

2. **移除操作**（line 172-177）
```python
# 修复前
for i, it in enumerate(data.get(dim, [])):
    data[dim].pop(i)

# 修复后
for i, it in enumerate(candidates.get(dim, [])):
    candidates[dim].pop(i)
```

3. **清空统计**（line 192-196）
```python
# 修复前
total = sum(len(data.get(d, [])) for d in [...])

# 修复后
total = sum(len(candidates.get(d, [])) for d in [...])
```

## 验证结果

  - 2.0板块卡位：5只

## 结论

修复成功，买点监控现在能正确读取候选池并执行日内移出逻辑。
