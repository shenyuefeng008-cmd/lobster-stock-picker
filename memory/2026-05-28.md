## ~15:17 涨跌家数采样脚本验证失败 - 数据严重不准确

### 实验结果
调用 `get_market_sentiment.py` 后返回：
- 上涨：~5322家
- 下跌：~0家
- 平盘：~178家

### 与实际指数严重矛盾
三大指数全部下跌：上证-0.22%、深证-0.74%、创业板-0.84%

### 根因
采样策略有致命BUG——股票代码不是用真实代码列表，而是顺序生成：
```python
for i in range(600000, 606000, 1000):
    sample_codes.append(f"sh{i:06d}")
```
生成 `sh600000`, `sh601000`, `sh602000`... 但大部分代码不存在或非活跃股，腾讯API返回无效数据导致结果完全失真。

### 修复方向
需要先获取真实股票代码列表再采样，而非自行构造代码序列。

### 后续待办
- 重新设计采样算法（基于真实代码列表抽样）
- 或改用其他可靠涨跌家数数据源
