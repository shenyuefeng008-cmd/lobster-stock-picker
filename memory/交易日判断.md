# 交易日判断规则

## 判断逻辑

### 方法一：星期判断（基础）
- 周一至周五：交易日
- 周六、周日：非交易日

### 方法二：API验证（精确）
调用腾讯 `qt.gtimg.cn` 接口，如果返回有效数据则为交易日，否则为非交易日。

```python
import akshare as ak
# 尝试获取当日数据，如果失败则为非交易日
try:
    df = ak.stock_zh_a_spot_em()
    if df.empty:
        IS_TRADING_DAY = False
    else:
        IS_TRADING_DAY = True
except:
    IS_TRADING_DAY = False
```

## 使用场景

所有 cron 任务执行前，先判断是否是交易日：
- 如果是交易日 → 正常执行
- 如果是非交易日 → 跳过执行，记录日志

## 当前实现

心跳系统 P0 前置检查：
1. 先检查星期（周一至周五）
2. 再调用 API 验证（qt.gtimg.cn）
3. 如果 API 返回有效数据 → 是交易日
4. 如果 API 无数据或报错 → 非交易日，跳过

## 注意事项

- 节假日调休（如国庆调休到周末）需要手动处理
- 临时休市（如疫情、极端天气）需要手动处理
- 建议结合多种方法判断，提高准确性
