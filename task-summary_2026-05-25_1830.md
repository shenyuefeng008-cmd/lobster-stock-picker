# 评分权重进化回路建成

## 新增的能力

**从"调外围参数"升级到"调评分因子权重"**

现在进化系统能追踪**每只候选股的每项评分因子**→等平仓后→**反推哪些因子有效/无效**→**自动调权重**。

## 改了什么

| 文件 | 改动 |
|------|------|
| `lobster_premarket_engine.py` | 4个维度的选股函数输出 `score_detail`（如`{"额得分":20, "板块强度得分":15}`），写入盘前JSON |
| `scoring_calculator.py` | 新增 `calculate_score_structured()` 输出分项得分JSON |
| `scripts/score_tracker.py` | **新建** — 读盘前JSON的 `score_detail` → 追加到 `trading/score_history.json` |
| `evolution_feedback_analyzer.py` | 新增 `_link_trades_to_factors()` + `_adjust_factor_weights()` — 关联平仓交易的PnL到分项得分，反向调权重 |

## 数据链路

```
盘前引擎 07:00
  ↓ 每个候选股带 score_detail
trader → 买入 → 卖出 → PnL 已知
  ↓
进化分析器 00:10
  ↓ 联立模拟持仓.trade_log + score_history.json
  分项得分 vs PnL -> 胜者因子加权分 - 败者因子加权分
  ↓ diff>3 → 权重+15%（此因子有效）
  ↓ diff<-1 → 权重-15%（此因子反向）
  ↓ abs(diff)<0.5 → -2%（无区分度）
  ↓
写入 lobster-config.json 的 scoring_models 权重
```

## 当前状态

- 数据链路已通，但 `score_history.json` 为空（因为今天的盘前数据还没score_detail字段——下次盘前引擎运行时会自动带上）
- 下次交易日 07:00 盘前 → score_tracker → 事后进化开始生效
- 需至少 3 笔同维度平仓且有 score_detail 记录，才会触发权重调整

## 下阶段（等数据积累）

先跑一个月。到时候 `score_history.json` 积累 30+ 条记录后，权重进化才有统计意义。
