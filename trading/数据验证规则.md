| 项目 | 错误值 | 正确值 | 错误类型 |
|------|--------|--------|----------|
| 股票 | 中巨芯-U(688549) | - | - |

- Agent看到涨幅10.90%，误以为科创板股票=20cm涨停

- 复盘报告称"20cm涨停，明日若高开回落可低吸"
- 实际只是普通大涨，未封板，明日策略错误

## ✅ 数据验证规则（强制执行）

### 规则1：涨停判断必须用价格验证

```python
def is_zhangting(现价, 昨收, 涨跌幅限制=0.1):
    """
    判断是否涨停（必须用价格，不能只看涨幅%）
    
    参数:
        现价: 当前价格
        昨收: 昨日收盘价
        涨跌幅限制: 默认10%，科创板/创业板注册制为20%
    
    返回:
        (是否涨停, 涨停类型)
    """
    涨停价 = 昨收 * (1 + 涨跌幅限制)
    
    # 允许0.5%误差（盘中可能略超涨停价）
    if 现价 >= 涨停价 * 0.995:
        if 涨跌幅限制 >= 0.19:
            return True, "20cm涨停"
        else:
            return True, "涨停"
    else:
        return False, "未涨停"
```

**强制要求**：
- ❌ 禁止只看涨幅%判断涨停（如"涨幅>9.9%则涨停"）
- ✅ 必须计算涨停价 = 昨收 × (1 + 涨跌幅限制)
- ✅ 必须验证 现价 ≥ 涨停价 × 0.995

### 规则2：涨跌幅限制必须按板块区分

```python
def get_zhangting_limit(代码):
    """
    根据代码判断涨跌幅限制
    
    返回:
        涨跌幅限制 (0.1=10%, 0.2=20%, 0.3=30%, None=无限制)
    """
    # 科创板（688开头）+ 创业板注册制（300开头且上市>5日）
    if 代码.startswith('688'):
        return 0.2  # 20%
    
    # 创业板（300开头）
    if 代码.startswith('300'):
        # 需查询上市日期，若上市>5日则20%，否则无限制
        # 简化处理：默认20%
        return 0.2
    
    # 北交所（8开头或4开头）
    if 代码.startswith('8') or 代码.startswith('4'):
        return 0.3  # 30%
    
    # 主板/中小板
    return 0.1  # 10%
```

**涨跌幅限制表**：

| 板块 | 代码特征 | 涨跌幅限制 | 备注 |
|------|----------|------------|------|
| 主板 | 60/000/001/002/003开头 | 10% | - |
| 科创板 | 688开头 | 20% | 上市前5日无限制 |
| 创业板 | 300开头 | 20% | 上市前5日无限制 |
| 北交所 | 8/4开头 | 30% | - |
| ST股 | ST/*ST开头 | 5% | - |

### 规则3：关键数据必须二次验证

**必须二次验证的数据**：

| 数据类型 | 第一次来源 | 第二次验证 | 验证方式 |
|----------|------------|------------|----------|
| 涨停判断 | akshare/Agent分析 | 腾讯qt.gtimg.cn | 现价 vs 涨停价 |
| 涨跌家数 | legulegu.com | - | 单源，但需检查日期 |
| 指数涨跌 | 腾讯qt.gtimg.cn | - | 单源，已验证 |
| 连板数 | akshare涨停池 | - | 单源，已验证 |

**验证代码示例**：

```python
# 涨停二次验证（必须执行）
def verify_zhangting(代码, Agent判断结果):
    # 从腾讯接口获取实时数据
    data = curl_qtimg(代码)
    现价 = data['现价']
    昨收 = data['昨收']
    
    # 计算涨跌幅限制
    limit = get_zhangting_limit(代码)
    涨停价 = 昨收 * (1 + limit)
    
    # 判断
    实际涨停 = 现价 >= 涨停价 * 0.995
    
    # 对比Agent判断
    if Agent判断结果 == "涨停" and not 实际涨停:
        print(f"⚠️ 数据错误：{代码} Agent判断涨停，实际未涨停")
        return False, "未涨停"
    elif Agent判断结果 == "未涨停" and 实际涨停:
        print(f"⚠️ 数据错误：{代码} Agent判断未涨停，实际涨停")
        return True, "涨停"
    else:
        return 实际涨停, Agent判断结果
```

## 🔧 代码修复示例（CRON_CLOSING_TASK.md）

**在步骤2涨停池数据获取后，加入验证**：

```python
# 涨停二次验证（新增）
def verify_zhangting_batch(stock_list):
    """
    批量验证涨停判断
    
    参数:
        stock_list: [(代码, 名称, Agent判断), ...]
    
    返回:
        [(代码, 名称, 正确判断, 是否修正), ...]
    """
    results = []
    
    # 批量获取实时数据
    codes = ','.join([f"sh{s[0]}" if s[0].startswith('6') else f"sz{s[0]}" for s in stock_list])
    data = curl_qtimg(codes)
    
    for 代码, 名称, Agent判断 in stock_list:
        if 代码 not in data:
            results.append((代码, 名称, Agent判断, False))
            continue
        
        现价 = data[代码]['现价']
        昨收 = data[代码]['昨收']
        limit = get_zhangting_limit(代码)
        涨停价 = 昨收 * (1 + limit)
        
        实际涨停 = 现价 >= 涨停价 * 0.995
        
        if Agent判断 == "涨停" and not 实际涨停:
            print(f"⚠️ 修正：{名称}({代码}) Agent判断涨停，实际未涨停")
            results.append((代码, 名称, "未涨停", True))
        elif Agent判断 == "未涨停" and 实际涨停:
            print(f"⚠️ 修正：{名称}({代码}) Agent判断未涨停，实际涨停")
            results.append((代码, 名称, "涨停", True))
        else:
            results.append((代码, 名称, Agent判断, False))
    
    return results
```
