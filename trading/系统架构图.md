## 1. 配置中心：lobster-config.json

所有可控参数**唯一入口**，禁止脚本内硬编码阈值。

```
lobster-config.json (v1.3)
├── _meta (版本/更新时间)
├── emotion (5段情绪阈值·仓位规则)
├── 1.0_first_to_second (一进二参数)
├── 1.0_divergence (分歧低吸参数)
├── 2.0_sector (板块卡位参数)
├── 3.0_trend (趋势低吸参数)
├── 3.0_emotion_rules ← NEW: 辅助模式低吸解锁规则
├── catalyst (五维评分权重/等级阈值)
├── stop_loss (三维度止损规则)
├── logic_disprove (逻辑证伪判定)
├── 3.0_holding_rules (持有期硬约束)
├── 3.0_exit_rules (出池阈值)
├── ice_freeze (冰点修复确认)
├── tracks (11大赛道列表)
├── trend_pool (容量池种子股+代码+硬约束)
├── scoring_models (四维度打分模型权重/阈值)
└── bid_filter_thresholds (四维度竞价过滤参数)
```

**读取者** (7个脚本):
`catalyst_scoring.py` `lobster_bid_filter_v2.py` `lobster_buypoint_detector.py`
`lobster_premarket_engine.py` `lobster_trend_pool_updater.py`
`scoring_calculator.py` `trading_decision_engine.py`

**规则**: 新增参数必须入config，脚本内不写死阈值。进化任务修改config而非改代码。

## 5. 情绪判定体系（三源保活）

```
                    实时情绪刷新链
                    ┌─────────────────┐
 07:00 盘前引擎 ───→│ emotion.get()   │ ← 盘前legulegu查询（已过时）
                    │ (老数据)        │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
 10:00-14:30       │ fetch_live_      │  每次买点扫描都刷新
 买点扫描每30分 ──→│ emotion()       │
                    │                  │
                    ├─ 主源 legulegu ─┤  (~1s) ✅
                    ├─ 备源 新浪5页  ─┤  (~3s) ✅
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  有实时数据?     │
                    │     ↓           │
                    │  Yes → 正常判断 │
                    │  No  → up=-1   │
                    │        暂停买点  │
                    └─────────────────┘
```

关键原则：
- **不缓存旧数据**：每次扫描都重新请求，拒绝fallback到09:45/07:00数据
- **双源独立**：legulegu挂了走新浪，新浪挂了走legulegu
- **全停保守**：两个源都挂 → 不触发任何买点

## 7. 数据管理规则

1. **配置** → lobster-config.json (唯一)
2. **持久数据** → trading/*.json (模拟盘、反馈、错误、催化)
3. **盘中中间数据** → /tmp/*.json (当日有效，次日清除)
4. **参考数据** → trading/*.md (日历、池子、日志，人工维护)
5. **数据同步** → 数据写入后必须同步所有副本（已修复: sell→同步/tmp缓存）
6. **非交易时段** → 禁止使用早于当前时刻的旧数据做决策
