### 第二步：理解三维度规则（15分钟）

**核心文件**：`lobster-rules.md`（v2.1）

**三维度框架**：
| 维度 | 名称 | 核心逻辑 | 适用情绪 |
|------|------|---------|---------|
| **1.0** | 点（打板） | 一进二 + 分歧低吸 | <1500 主导 |
| **2.0** | 线（板块轮动） | 板块卡位 + 高低切 | 2500-3500 主导 |
| **3.0** | 面（容量趋势） | 产业逻辑 + 趋势低吸 | 1500-2500 辅助 |

**维度切换规则**（根据涨跌家数）：
- `<1500` → 1.0 主导，仓位上限 30%
- `1500-2500` → 1.0 + 3.0，总仓位上限 90%
- `2500-3500` → 2.0 + 1.0，5-7 成
- `>3500` → 辅助模式（一字板套利或空仓），1-2 成

### 第三步：理解数据流（10分钟）

```
腾讯 qt.gtimg.cn  → 指数/个股实时涨跌（✅ 主数据源）
legulegu.com      → 涨跌家数（✅ 核心情绪数据）
akshare          → 涨停池/炸板池/历史K线（✅）
Playwright       → 东方财富网页抓取（✅ 备用）
```

**⚠️ 关键教训**：
- **实时涨跌** → 必须用腾讯 `qt.gtimg.cn`（akshare K线延迟到上一交易日）
- **均线计算** → 用 akshare K线（历史数据准确）
- **涨跌家数** → 用 legulegu.com（`web_fetch` 工具会被 SSRF 拦截，必须用 `curl`）
