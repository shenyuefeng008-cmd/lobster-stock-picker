- **现象**：`lobster_trend_pool_updater.py` 执行时间过长被终止
- **影响**：趋势容量池仍为2026-06-05版本（8只标的），未反映最新市场变化
- **根因**：脚本遍历37只种子股×多接口调用，未设置单个请求超时

### 修复方案
修改 `scripts/lobster_trend_pool_updater.py`：

1. **添加全局超时控制**（120秒）：
   ```python
   import signal
   
   SCRIPT_TIMEOUT = 120
   
   def timeout_handler(signum, frame):
       print("\n⚠️ 脚本执行超时，强制退出")
       sys.exit(1)
   
   signal.signal(signal.SIGALRM, timeout_handler)
   signal.alarm(SCRIPT_TIMEOUT)
   ```

2. **添加单请求超时控制**（15秒）：
   ```python
   REQUEST_TIMEOUT = 15
   
   # 修改 get_tencent_kline：
   r = subprocess.run(["curl", "-s", "--max-time", str(REQUEST_TIMEOUT), url],
                      capture_output=True, timeout=REQUEST_TIMEOUT + 2)

   r = subprocess.run(["curl", "-s", "--max-time", str(REQUEST_TIMEOUT), url],
                      capture_output=True, timeout=REQUEST_TIMEOUT + 2)
   ```

3. **添加异常处理**：
   ```python

           # ... 原有逻辑 ...
   ```

4. **单股超时控制**：
   ```python
   for name, code in STOCK_CODE_MAP.items():
       try:
           # 设置单只股票处理超时
   ```
