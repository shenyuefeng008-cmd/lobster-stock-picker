# 任务总结 2026-05-28 13:55

## 问题
情绪判断出错 — 涨跌家数数据错误导致情绪误判

## 根本原因
`scripts/get_market_sentiment.py` 中 `get_market_sentiment_legulegu()` 函数有 bug：

```python
# 错误代码
down = int(md.group(2)) if md else 0  # ❌ 正则只有1个group，但引用了group(2)
```

正则 `r'(\d+)家下跌'` 只有一个捕获组 → `group(1)` 是正确写法，`group(2)` 触发 `IndexError: no such group`

## 错误链条
1. `get_market_sentiment_legulegu()` 异常 → 返回 `(-1,-1,-1,-1)`
2. 回退到腾讯采样 → 停牌股（涨跌幅=0+成交量=0）被计入平盘 → **down=0, up=5500**
3. 情绪 → `5500涨/0跌` → **极度高潮（辅助模式）** ❌
4. 实际 legulegu 数据：`2945涨/2122跌` → **高潮期（2.0+1.0，7成）** ✅

## 修复方案
1. **修复 group(1) 索引**：`md.group(1)` 而非 `md.group(2)`
2. **主备源重构**：legulegu 为主源（精确），腾讯采样降为备源
3. **腾讯采样加强**：扩大采样范围（每50→116只），过滤停牌股

## 验证结果
```
修复前：5500涨/0跌 → 极度高潮（辅助模式）❌
修复后：2945涨/2122跌 → 高潮期（2.0+1.0，7成）✅
数据源：legulegu.com（直接数据，非采样估算）
涨停109家，跌停10家，checked=5067只
```
