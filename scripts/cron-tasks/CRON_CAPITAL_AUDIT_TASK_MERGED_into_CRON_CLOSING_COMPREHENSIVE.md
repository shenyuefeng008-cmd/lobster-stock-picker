# CRON_CAPITAL_AUDIT_TASK.md — 资金账本自动复核

## 任务信息

- **任务名**: 资金账本自动复核
- **执行时间**: 每个交易日 15:30
- **Cron表达式**: `30 15 * * 1-5`
- **超时**: 60秒
- **版本**: v1.0（2026-06-17）

---

## 任务目标

每天收盘后自动核验 `trading/模拟持仓.json` 的 `capital` 字段是否准确：
1. 检查 `trade_log` 是否有透支卖出（SELL股数 > 之前BUY累计）
2. 逐笔FIFO重新计算可用现金
3. 对比 `capital` 字段，发现差异自动修正
4. 结果写入 `trading/capital_audit_YYYYMMDD.log`

---

## 执行步骤

### 1. 运行复核脚本

```bash
python3 scripts/validate_capital.py
```

**预期输出**：
- 无透支卖出 → 继续
- 有透支卖出 → 记录到 `trading/BUG_LOG.md`
- capital字段准确 → 完成
- capital字段有差异 → 自动修正并写回文件

### 2. 读取审计日志

```bash
cat trading/capital_audit_$(date +%Y%m%d).log
```

### 3. 推送结果到消息

**如果无差异**：
```
✅ 资金账本复核通过
总资产：¥{total_assets:,.0f}
累计盈亏：{total_pnl:+,.0f} ({total_pnl_pct:+.1f}%)
```

**如果发现差异并修正**：
```
⚠️ 资金账本已自动修正
差异原因：{error_details}
修正前：¥{old_total:,.0f}
修正后：¥{new_total:,.0f}
差异：{diff:+,.0f}
```

---

## 错误处理

- 脚本执行失败 → 推送错误日志，不阻塞
- 模拟持仓.json不存在 → 跳过，推送告警
- 审计日志写入失败 → 推送告警
