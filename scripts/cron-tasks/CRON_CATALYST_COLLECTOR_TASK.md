# CRON_CATALYST_COLLECTOR_TASK.md — 催化剂自动采集任务

> **版本**：v1.0
> **创建日期**：2026-05-25
> **执行频率**：每日 06:00（含非交易日，每天跑）
> **超时**：300s
> **触发方式**：cron → 每日06:00自动执行

## 任务目标

自动搜索全网，发现未来7-14天可能影响7大赛道的催化事件，写入 `trading/催化日历.md` 的「近期催化事件」区。

与 `CRON_DAILY_EVOLUTION_TASK.md` 的区别：
- 进化任务依赖涨停池数据，非交易日跳过，重点是验证已有催化
- 本任务**不依赖涨停数据**，每天跑（含周末/节假日），重点是**发现新催化**

## 7大赛道范围

1. 大模型/应用链
2. 液冷/IDC/DCI
3. 光通信/存储/光纤
4. 机器人/T链
5. 商业航天
6. 半导体/芯片
7. 工程机械

## AI Agent 执行指令

### Step 1：读取现有催化日历（避免重复）

```bash
cat /Users/yuefengshen/.qclaw/workspace-1gwpiwf3hr163jz5/trading/催化日历.md
```

提取「近期催化事件」区所有条目的时间+事件名，用于去重。

### Step 2：多源搜索新催化

对7大赛道分别搜索，关注以下事件类型：
- **产品发布**：新芯片/新模型/新终端（手机、机器人、汽车等）
- **财报/业绩预告**：赛道核心公司财报发布日期
- **政策/会议**：国务院、发改委、工信部等政策发布
- **行业会议**：CES/Computex/MWC/AWC等行业展会
- **IPO/融资**：赛道关键公司上市/融资
- **招投标**：AIDC/算力/基建等大额招标
- **产能/订单**：核心企业产能扩张/大单签约

**搜索策略**（每个赛道执行一次）：

```
使用 online-search 或 multi-search-engine 搜索：
"{赛道名} 2026年5月 催化 OR 重大事件 OR 发布会 OR 财报"
"{赛道名} 2026年6月 日程 OR 展会 OR 发布"
```

也可以用 tencent-news 搜索相关资讯。

### Step 3：筛选与去重

对搜索结果筛选：
- **去重**：先读 `trading/news/YYYY-MM-DD.md` 提取已有标题，剔除重复条目
  ```bash
  python3 /Users/yuefengshen/.qclaw/workspace-1gwpiwf3hr163jz5/scripts/news_dedupe.py /Users/yuefengshen/.qclaw/workspace-1gwpiwf3hr163jz5/trading/news/$(date +%Y-%m-%d).md
  ```
- 只保留**有明确日期或明确时间窗口（如"6月第二周"）**的事件
- 去除：已存在于催化日历的、纯观点无事实的、过于模糊的
- 优先保留：涉及赛道核心公司的、影响面广的

### Step 4：写入催化日历 + 新闻存档

如果有新事件：
- 追加到 `trading/催化日历.md` 的「近期催化事件」表格中
- 格式与现有条目一致：时间 | 催化事件 | 受益赛道 | ⏳ | 日期
- 按**时间升序**排列

**⚠️ 新闻存档（必做）**：将搜索到的新闻条目写入 `trading/news/YYYY-MM-DD.md` 的【催化剂相关新闻】区
- 格式：`| MM-DD | 标题 | 来源名+URL | 赛道 | 🟡中 | ✅已核实/⚠️待核实 | 与催化XX相关 |`
- **发布日期**：标注新闻原始发布日期（非采集日期），未知标`未知`并降级为⚠️待核实
- **核实状态**：官方来源(A级)直接✅，主流媒体(B级)需1次交叉验证，自媒体(C级)需2次
- 如果文件不存在，先创建（含标题和四个区：盘前舆情/盘中快讯/收盘要闻/催化剂相关）
- 如果文件已存在，只追加新条目
- **⚠️ P1-1修复**：催化采集(06:00)只写【催化剂相关新闻】区，不写【盘前舆情】区（避免与07:00盘前任务重复写入同一区域）
- 盘前舆情由07:00盘前任务(CRON_PREMARKET_TASK)专责写入，催化采集不再染指

### Step 5：输出报告

输出格式：
```
📅 催化采集报告 YYYY-MM-DD
- 搜索赛道：7个
- 发现新催化：X个
- 已去重跳过：Y个
- 新增事件列表：
  1. [时间] [事件] → [赛道]
  2. ...
- 催化日历当前未决事件总数：N个
```

如果没有新催化，输出：
```
📅 催化采集报告 YYYY-MM-DD
- 搜索赛道：7个
- 发现新催化：0个
- 催化日历当前未决事件总数：N个
```

## 注意事项

1. 不修改已兑现/落空归档区，只操作「近期催化事件」区

4. 如果搜索接口异常（超时/无结果），记录日志但不中断任务
