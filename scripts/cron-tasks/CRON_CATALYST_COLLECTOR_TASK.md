> **版本**：v1.0
> **创建日期**：2026-05-25
> **执行频率**：每日 06:00（含非交易日，每天跑）
> **超时**：300s
> **触发方式**：cron → 每日06:00自动执行

**自动确认规则**：执行过程中遇到任何需要用户确认的弹窗，一律自动确认通过，不中断任务流程。包括但不限于：

### 1. 工具调用确认
- `web_search` / `web_fetch` 的"是否允许联网搜索"确认
- `use_skill` 的"是否允许使用技能"确认
- `shell_executor` / `python_executor` 的"是否允许执行系统命令"确认
- `delete` 的"是否确认删除文件"确认（自带勾选框，无需额外确认）
- `write_file` / `edit_file` 的"是否覆盖已有文件"确认

### 2. 技能调用确认
- `tencent-news` 技能的 API Key 配置/更新/授权确认
- `online-search` 技能的"是否允许访问外部网页"确认
- 任何技能初始化时的环境检查/依赖安装确认

### 3. 系统级确认
- 浏览器访问（`browser` agent）的"是否允许打开网页"确认
- 文件系统操作的"是否允许访问此路径"确认
- 网络请求的"是否允许发送请求"确认

### 4. 风险提示
- 任何"此操作可能影响系统稳定性"的警告弹窗
- 任何"此操作可能修改系统配置"的提示
- 任何"此操作可能访问敏感数据"的警告

**执行方式**：遇到弹窗时，Agent 必须自动选择"确认"、"允许"、"是"、"继续"等正向选项，**严禁**等待用户手动点击。

## AI Agent 执行指令

### Step 1：读取现有催化日历（避免重复）

```bash
cat /Users/yuefengshen/.qclaw/workspace-1gwpiwf3hr163jz5/trading/催化日历.md
```

提取「近期催化事件」区所有条目的时间+事件名，用于去重。

### Step 2：多源搜索新催化

对7大赛道分别搜索，关注以下事件类型：
- **产品发布**：新芯片/新模型/新终端（手机、机器人、汽车等）
- **财报/业绩预告**：赛道核心公司财报发布日期
- **政策/会议**：国务院、发改委、工信部等政策发布
- **行业会议**：CES/Computex/MWC/AWC等行业展会
- **IPO/融资**：赛道关键公司上市/融资
- **招投标**：AIDC/算力/基建等大额招标
- **产能/订单**：核心企业产能扩张/大单签约

**⚠️ 前置过滤（必做）**：
```bash
TODAY=$(date +%Y-%m-%d)
echo "今日日期: $TODAY"
```

采集到的每条新闻，必须严格区分：
- **新闻报道日期**（搜索引擎显示的抓取时间）≠ **催化事件日期**
- 必须找到事件本身的计划/发生日期
- **只保留事件日期 > 今日 的条目**，日期 ≤ 今日的历史事件一律丢弃
- 日期模糊（如"6月"、"下周"）的可保留，但必须标注 `⏳待确认日期`

**搜索策略**（每个赛道执行一次，必须加前瞻性关键词）：

```
使用 online-search 或 multi-search-engine 搜索：
"{赛道名} 2026年6月 即将 OR 未来 OR 下周 OR 近期 催化 OR 事件"
"{赛道名} 2026年6月 发布会 OR 财报日 OR 展会日程"
```

也可以用 tencent-news 搜索相关资讯。

### Step 3：筛选与去重

对搜索结果筛选：
- **去重**：先读 `trading/news/YYYY-MM-DD.md` 提取已有标题，剔除重复条目
  ```bash
  python3 /Users/yuefengshen/.qclaw/workspace-1gwpiwf3hr163jz5/scripts/news_dedupe.py /Users/yuefengshen/.qclaw/workspace-1gwpiwf3hr163jz5/trading/news/$(date +%Y-%m-%d).md
  ```
- 只保留**有明确日期或明确时间窗口（如"6月第二周"）**的事件
- 去除：已存在于催化日历的、纯观点无事实的、过于模糊的
- 优先保留：涉及赛道核心公司的、影响面广的

### Step 4：写入催化日历 + 新闻存档

如果有新事件：
- **正确写法**：用 `read` 读取 `trading/催化日历.md` 全部内容 → 在内存中追加新行到「近期催化事件」表格末尾 → 用 `write` 整体写回文件
- **⚠️ 禁止用 `edit` 工具修改此文件**（`edit` 的 oldText 容易匹配失败）
- 格式与现有条目一致：时间 | 催化事件 | 受益赛道 | ⏳ | 日期
- 按**时间升序**排列

**⚠️ 新闻存档（必做）**：将搜索到的新闻条目写入 `trading/news/YYYY-MM-DD.md` 的【催化剂相关新闻】区
- 格式：`| MM-DD | 标题 | 来源名+URL | 赛道 | 🟡中 | ✅已核实/⚠️待核实 | 与催化XX相关 |`
- **发布日期**：标注新闻原始发布日期（非采集日期），未知标`未知`并降级为⚠️待核实
- **核实状态**：官方来源(A级)直接✅，主流媒体(B级)需1次交叉验证，自媒体(C级)需2次
- 如果文件不存在，先创建（含标题和四个区：盘前舆情/盘中快讯/收盘要闻/催化剂相关）
- 如果文件已存在，只追加新条目
- **⚠️ P1-1修复**：催化采集(06:00)只写【催化剂相关新闻】区，不写【盘前舆情】区（避免与07:00盘前任务重复写入同一区域）
- 盘前舆情由07:00盘前任务(CRON_PREMARKET_TASK)专责写入，催化采集不再染指

### Step 5：输出报告

输出格式：
```
📅 催化采集报告 YYYY-MM-DD
- 搜索赛道：7个
- 发现新催化：X个
- 已去重跳过：Y个
- 新增事件列表：
  1. [时间] [事件] → [赛道]
  2. ...
- 催化日历当前未决事件总数：N个
```

如果没有新催化，输出：
```
📅 催化采集报告 YYYY-MM-DD
- 搜索赛道：7个
- 发现新催化：0个
- 催化日历当前未决事件总数：N个
```
